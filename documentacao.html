CopiMail

Sistema de Migração de Emails - Documentação Completa

Visão Geral do Sistema

O CopiMail é um sistema automatizado de migração de emails que permite transferir mensagens entre diferentes servidores de email de forma segura e eficiente.

🔐 Segurança

Suporte a autenticação segura e conexões criptografadas para proteger seus dados durante a migração.

⚡ Performance

Processamento otimizado para migrações rápidas, mesmo com grandes volumes de emails.

🔄 Automatização

Processo totalmente automatizado que requer apenas configuração inicial.

📊 Monitoramento

Acompanhamento em tempo real do progresso da migração com barras de progresso visuais.

Arquitetura do Sistema

O CopiMail utiliza a biblioteca padrão imaplib do Python para conectar aos servidores IMAP, garantindo compatibilidade com a maioria dos provedores de email.

Componentes Principais:
  • copimail.py - Script principal de migração
  • install_copimail.py - Instalador automático
  • requirements.txt - Dependências Python
  • version.json - Metadados do sistema

Instalação do Sistema

Requisitos do Sistema

  • Python 3.6+ instalado no sistema
  • Conexão com internet para download dos arquivos
  • Permissões de administrador (para configuração do PATH no Windows)

Instalação Automática

O CopiMail possui um instalador automático que configura todo o sistema em poucos passos:

1 Download do instalador: Baixe o arquivo install_copimail.py do repositório GitHub.
curl -O https://raw.githubusercontent.com/Samuel-Jordesson/copimail/main/install_copimail.py
2 Executar o instalador: Execute o script de instalação com Python.
python install_copimail.py
3 Confirmação: Confirme a instalação digitando 's' quando solicitado.
4 Aguarde: O sistema será baixado e configurado automaticamente com barra de progresso.

O que o Instalador Faz

Etapa Descrição Progresso
Download Baixa os arquivos do sistema do GitHub 0% - 33%
Dependências Instala bibliotecas Python necessárias 33% - 67%
Configuração Cria comandos globais e configura PATH 67% - 100%
✅ Configuração Automática do PATH: No Windows, o instalador configura automaticamente a variável de ambiente PATH, eliminando a necessidade de configuração manual.

Verificação da Instalação

Após a instalação, reinicie o terminal e execute:

copimail --help

Se o comando for reconhecido, a instalação foi bem-sucedida!

Como Funciona o Sistema

Princípio de Funcionamento

O CopiMail funciona como um ponte de migração entre servidores de email, utilizando o protocolo IMAP para acessar e transferir mensagens.

Fluxo de Migração:
  1. Conecta ao servidor de origem (IMAP)
  2. Autentica com credenciais fornecidas
  3. Lista todas as pastas de email
  4. Conecta ao servidor de destino (IMAP)
  5. Autentica no servidor de destino
  6. Transfere mensagens pasta por pasta
  7. Mantém estrutura de pastas original

Arquitetura Técnica

# Estrutura básica do sistema ├── Conexão IMAP (Origem) ├── Autenticação e Listagem ├── Processamento de Pastas ├── Conexão IMAP (Destino) ├── Criação de Pastas └── Transferência de Mensagens

Protocolos Suportados

  • IMAP4 - Protocolo principal para acesso a emails
  • SSL/TLS - Criptografia para conexões seguras
  • Autenticação - Login e senha padrão

Tratamento de Erros

O sistema possui tratamento robusto de erros para garantir que a migração continue mesmo em caso de problemas:

  • Reconexão automática em caso de falha de rede
  • Retry inteligente para mensagens com problemas
  • Log detalhado de todas as operações
  • Recuperação de estado para migrações interrompidas

Como Usar o Sistema

Execução Básica

Após a instalação, o CopiMail pode ser executado de duas formas:

# Forma 1: Comando global (recomendado) copimail # Forma 2: Execução direta do Python python ~/copimail/copimail.py

Interface Interativa

O sistema apresenta uma interface de linha de comando intuitiva:

1 Banner de boas-vindas com informações do sistema
2 Configuração do servidor de origem (IMAP, porta, usuário, senha)
3 Configuração do servidor de destino (IMAP, porta, usuário, senha)
4 Confirmação das configurações antes de iniciar a migração
5 Execução da migração com barras de progresso em tempo real

Exemplo de Configuração

=== CONFIGURAÇÃO DO SERVIDOR DE ORIGEM === Servidor IMAP: imap.gmail.com Porta: 993 Usuário: seu.email@gmail.com Senha: ******** === CONFIGURAÇÃO DO SERVIDOR DE DESTINO === Servidor IMAP: imap.outlook.com Porta: 993 Usuário: seu.email@outlook.com Senha: ********

Barras de Progresso

Durante a migração, o sistema exibe múltiplas barras de progresso:

  • Progresso geral da migração completa
  • Progresso por pasta sendo processada
  • Progresso de mensagens na pasta atual
  • Estatísticas em tempo real (mensagens processadas, tempo estimado)

Configuração e Personalização

Configurações de Rede

O sistema suporta diferentes configurações de rede para se adaptar a diversos ambientes:

Configuração Padrão Descrição
Timeout de conexão 30 segundos Tempo máximo para estabelecer conexão
Timeout de operação 60 segundos Tempo máximo para operações IMAP
Tentativas de reconexão 3 Número de tentativas em caso de falha
Delay entre tentativas 5 segundos Espera entre tentativas de reconexão

Configurações de Segurança

⚠️ Importante: Sempre use conexões SSL/TLS para proteger suas credenciais e dados durante a migração.
  • SSL/TLS obrigatório para servidores que suportam
  • Verificação de certificados para garantir autenticidade
  • Criptografia de dados em trânsito

Configurações de Performance

Para otimizar a velocidade da migração:

  • Processamento em lotes de mensagens
  • Buffer de memória configurável
  • Limitação de taxa para evitar sobrecarga do servidor
  • Paralelização de operações quando possível

Solução de Problemas

Problemas Comuns e Soluções

1. Erro de Conexão IMAP

Sintoma: "Falha ao conectar ao servidor IMAP"
Solução: Verifique se o servidor e porta estão corretos, e se o firewall não está bloqueando a conexão.

2. Erro de Autenticação

Sintoma: "Credenciais inválidas"
Solução: Verifique usuário e senha. Para Gmail, pode ser necessário usar "senha de app" em vez da senha principal.

3. Comando 'copimail' não encontrado

Sintoma: "comando não é reconhecido"
Solução: Reinicie o terminal após a instalação, ou execute o script diretamente com Python.

4. Migração interrompida

Sintoma: Processo para no meio da migração
Solução: Execute novamente o comando. O sistema detectará automaticamente o ponto de parada e continuará de onde parou.

Logs e Diagnóstico

O sistema gera logs detalhados para facilitar o diagnóstico de problemas:

# Exemplo de log de migração [2024-01-15 10:30:15] Conectando ao servidor de origem... [2024-01-15 10:30:16] Autenticação bem-sucedida [2024-15 10:30:17] Listando pastas... [2024-01-15 10:30:18] Encontradas 15 pastas [2024-01-15 10:30:19] Iniciando migração da pasta "INBOX"...

Verificação de Conectividade

Para testar se os servidores estão acessíveis:

# Teste de conectividade com telnet telnet imap.gmail.com 993 telnet imap.outlook.com 993 # Teste com Python python -c "import imaplib; print('Conectando...'); imaplib.IMAP4_SSL('imap.gmail.com', 993)"

Suporte e Contato

Para problemas não resolvidos:

  • Verifique a documentação oficial
  • Consulte os logs de erro
  • Teste com servidores de email diferentes
  • Verifique as configurações de firewall e proxy
